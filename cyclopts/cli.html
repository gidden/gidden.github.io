Cyclopts Command Line Interface (CLI)Â¶

Running cyclopts -h results in

usage: Cyclopts [-h] {convert,exec,condor} ...

positional arguments:
  {convert,exec,condor}
    convert             Convert a parameter space defined by an input run control file into an HDF5
                        database for a Cyclopts execution run.
    exec                Executes a parameter sweep as defined by the input database and other command
                        line arguments.
    condor              Submits a job to condor, retrieves output when it has completed, and cleans up
                        the condor user space after.

optional arguments:
  -h, --help            show this help message and exit

Running cyclopts convert -h results in

usage: Cyclopts convert [-h] [-i INPUT] [-o OUTPUT]

optional arguments:
  -h, --help            show this help message and exit
  -i INPUT, --input INPUT
                        The run control file to use that defines a continguous parameter space.
  -o OUTPUT, --output OUTPUT
                        The HDF5 file to dump converted parameter space points to. This file can later
                        be used an input to an execute run.

Running cyclopts exec -h results in

usage: Cyclopts exec [-h] [-i INPUT] [-o OUTPUT] [--solvers [SOLVERS [SOLVERS ...]]] [--rc RC]

optional arguments:
  -h, --help            show this help message and exit
  -i INPUT, --input INPUT
                        An HDF5 file that defines points in the parameter space to be run.
  -o OUTPUT, --output OUTPUT
                        An HDF5 file to which output will be provided.
  --solvers [SOLVERS [SOLVERS ...]]
                        A list of which solvers to use..
  --rc RC               The run control file, which allows idetification of a subset of input to run.

Running cyclopts condor -h results in

usage: Cyclopts condor [-h] [-u USER] [-t HOST] [-i INDB] [-d DUMPDIR] [-o OUTDB] [--no-clean]
                       [--solvers [SOLVERS [SOLVERS ...]]] [--no-auth]

optional arguments:
  -h, --help            show this help message and exit
  -u USER, --user USER  The condor user name.
  -t HOST, --host HOST  The condor submit host.
  -i INDB, --input INDB
                        The input database.
  -d DUMPDIR, --dumpdir DUMPDIR
                        The directory in which to place output.
  -o OUTDB, --output OUTDB
                        The output database.
  --no-clean            Do *not* clean up the submit node after.
  --solvers [SOLVERS [SOLVERS ...]]
                        A list of which solvers to use on each run.
  --no-auth             Do not ask for a password for authorization.
